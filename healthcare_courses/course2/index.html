    üß† Course 2: Clinical Data ‚Äì Module-by-Module Mental Flows #

    A strategic mental map for each module using 5 guiding questions.

    üß≠ Module 1: Asking and Answering Questions via Clinical Data Mining #

    1. What’s the Problem?
      Clinicians have important questions but don’t know how to systematically answer them using data.

    2. Why Does It Matter?
      Better questions ‚Üí better care. Good questions drive data use and decision-making.

    3. Core Idea
      The 4-step data mining workflow:
      (1) Pose question ‚Üí (2) Find data ‚Üí (3) Transform data ‚Üí (4) Analyze

    4. How It Works
      Frame a question (e.g. SLE + clot risk), identify cohorts, extract patient timelines, compute outcomes.

    5. How It Connects
      Foundation for the entire course. All downstream steps depend on having a well-framed question.

    üè• Module 2: Data Available from Healthcare Systems #

    1. What’s the Problem?
      Clinical data is complex, biased, and scattered across different sources.

    2. Why Does It Matter?
      Understanding data provenance is essential for valid interpretation and modeling.

    3. Core Idea
      Types of healthcare data (structured, unstructured), who produces them, and common biases/errors.

    4. How It Works
      Learn data sources: EMRs, claims, pharmacies, registries. Understand missingness, selection bias, coding issues.

    5. How It Connects
      These data sources fuel patient timelines and features (Modules 3‚Äì4). Knowing their limitations is critical.

    üï∞Ô∏è Module 3: Representing Time in Clinical Data #

    1. What’s the Problem?
      Most clinical events happen over time, but databases don’t naturally express time relationships.

    2. Why Does It Matter?
      Misaligned event timing distorts exposures and outcomes ‚Äî ruins inference and prediction.

    3. Core Idea
      Use patient timelines. Choose timescales, bins, and encode order of events.

    4. How It Works
      Define index time (exposure), align with outcome, handle right-censoring, test for non-stationarity.

    5. How It Connects
      Prepares data for feature engineering (Module 4). Enables temporal reasoning in phenotyping (Module 6).

    üß± Module 4: Creating Analysis-Ready Datasets #

    1. What’s the Problem?
      Raw timeline data is too messy for analysis or ML.

    2. Why Does It Matter?
      Poorly engineered features ‚Üí invalid conclusions and unfair models.

    3. Core Idea
      Construct a patient-feature matrix. Engineer features from timeline + structured data.

    4. How It Works
      Clean ‚Üí normalize ‚Üí impute ‚Üí reduce ‚Üí create new features. Use knowledge graphs for semantics.

    5. How It Connects
      Structured input for downstream modeling and phenotyping. Builds on Module 3 and supports Module 6.

    üìÑ Module 5: Handling Unstructured Data (Text, Images, Signals) #

    1. What’s the Problem?
      Important clinical info is buried in unstructured formats like notes, scans, signals.

    2. Why Does It Matter?
      Ignoring this info causes mislabeling or missing key predictors.

    3. Core Idea
      Use NLP and signal/image processing to extract structured features.

    4. How It Works
      NLP ‚Üí detect negation/context ‚Üí extract features using knowledge graphs. Apply to text/images/signals.

    5. How It Connects
      Complements Module 4. Enables richer feature sets and more accurate phenotyping (Module 6).

    üß¨ Module 6: Electronic Phenotyping #

    1. What’s the Problem?
      We need to determine who really has the condition of interest, but it‚Äôs not always explicit in data.

    2. Why Does It Matter?
      Phenotyping determines cohort inclusion and outcome labeling ‚Äî critical for valid research.

    3. Core Idea
      Use rule-based or machine learning‚Äìbased methods to define presence of disease/condition.

    4. How It Works
      Define inclusion/exclusion criteria (rules), or train classifiers using imperfect labels and anchors.

    5. How It Connects
      Relies on timelines (3), features (4), and text (5). Prepares labeled cohorts for modeling or trial selection.

    ‚öñÔ∏è Module 7: Clinical Data Ethics #

    1. What’s the Problem?
      Using clinical data without safeguards risks privacy violations and public mistrust.

    2. Why Does It Matter?
      Ethics is essential for legal compliance, public trust, and fairness in AI.

    3. Core Idea
      Apply ethical frameworks (Belmont Report, Learning Health System) to govern data use and consent.

    4. How It Works
      Use informed consent, de-identification, appropriate return of results, and respect patient autonomy.

    5. How It Connects
      Applies across all modules. Ensures responsible use of all clinical data workflows and AI tools.

    Understand Data Sources & Biases (Module 2) ‚Üí
    Track Time and Events Correctly (Module 3) ‚Üí
    Transform into Clean Features (Module 4) ‚Üí
    Extract from Unstructured Formats (Module 5) ‚Üí
    Define Conditions Precisely (Module 6) ‚Üí
    Use Ethically & Fairly (Module 7)
