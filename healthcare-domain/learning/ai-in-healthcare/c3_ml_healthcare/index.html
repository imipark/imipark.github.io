    MODULE 3: Concepts and Principles of ML in Healthcare #

    1. What makes deep learning different from traditional ML?
    Deep learning organizes parameters into hierarchical layers allowing models to learn complex functions beyond input feature limits.

    2. How does a deep learning model transform data through layers?
    By repeatedly applying linear combinations and nonlinear activation functions (e.g., ReLU), the model generates meaningful features.

    3. What is a dense layer in neural networks?
    A dense or fully connected layer has neurons applying transformations to the same input, forming a building block of deep nets.

    4. What does the training loop in deep learning involve?
    Prediction → Loss calculation → Parameter update (gradient descent) → Validation → Hyperparameter tuning.

    5. How do different loss functions behave?

    • MSE penalizes large errors.
    • MAE treats all errors equally.
    • Cross-entropy is preferred for classification tasks.

    6. How does gradient descent work?
    It computes the gradient (slope) of the loss and adjusts model parameters to reduce the error.

    7. Why is backpropagation crucial?
    It enables computing gradients layer-by-layer from output to input, allowing efficient updates in deep networks.

    REPRESENTING UNSTRUCTURED DATA #

    8. How are images and text represented?

    • Images: Pixel grids, 0–255 intensity values.
    • Text: Word embeddings in high-dimensional space (300–1024 dims).

    9. Why use CNNs for image tasks?
    CNNs preserve spatial structure and use filters to detect visual patterns efficiently.

    10. How do CNN layers increase feature abstraction?
    Each layer has a larger receptive field, allowing detection of complex features from simple edges.

    11. What makes RNNs fit for NLP tasks?
    They handle sequential data by maintaining a context vector across time steps.

    12. How do Transformers improve over RNNs?
    They use self-attention to capture long-term dependencies in parallel, improving speed and accuracy.

    ADVANCED NETWORKS AND TASKS #

    13. What are key CNN architectures?

    • AlexNet: First with ReLU, 8 layers.
    • VGG: Deeper with small filters.
    • GoogLeNet/Inception: Parallel filter paths.
    • ResNet: Skip connections, 152+ layers.

    14. What are vision tasks beyond classification?

    • Semantic segmentation: Per-pixel classification.
    • Object detection: Bounding boxes.
    • Instance segmentation: Combines both.

    15. What is reinforcement learning (RL) and why is it hard in healthcare?
    RL needs simulatable environments and reward structures, both difficult in clinical settings.

    MODULE 4: Evaluation and Metrics for ML in Healthcare #

    16. How should we split data for evaluation?
    Typical splits: Train (70–80%), Validation, Test. Use k-fold cross-validation for small datasets.

    17. What is underfitting vs overfitting?

    • Underfitting: Too simple to learn patterns.
    • Overfitting: Memorizes training data noise.

    18. How do you detect fitting issues?
    Use learning curves to visualize loss and accuracy trends on train vs validation data.

    19. How to combat overfitting?
    Regularization (L1/L2, Dropout), data augmentation, simpler models.

    20. Why is accuracy misleading in imbalanced data?
    It may hide poor minority-class performance; use metrics like precision, recall, ROC/PR curves.

    21. What are ROC and PR curves?

    • ROC: TPR vs FPR across thresholds.
    • PR: Precision vs Recall; better for imbalanced datasets.

    MODULE 5: Strategies and Challenges in ML for Healthcare #

    22. Why is causation vs correlation important?
    ML often learns correlations that may not reflect true medical causes—leading to misleading outcomes.

    23. What are confounding variables in healthcare ML?
    Spurious patterns (e.g., gray hair as predictor of heart disease) that mislead models.

    24. What is the “black box” problem?
    Complex models lack interpretability. Trade-off exists between accuracy and explainability.

    25. How can we interpret black-box models?
    Use saliency maps, gradients, CAMs, and visualize neuron activations.

    26. What makes a model interpretable?

    • Transparent: Easily understood logic.
    • Explainable: Clear rationale for output.
    • Inspectable: Internally examinable.

    27. What is intrinsic vs post-hoc interpretability?

    • Intrinsic: Built-in simplicity (e.g., LACE index).
    • Post-hoc: Retrospective interpretation (e.g., heatmaps).

    DATA QUALITY AND BIAS CHALLENGES #

    28. Why is data quality so crucial?
    “Garbage in, garbage out.” Bad data → bad models. Data should be clean, recent, relevant, labeled.

    29. What is the class imbalance problem?
    Rare-event labels skew performance and metrics. Address via resampling, proper metrics, or specialized models.

    30. Does more data always help?
    Not necessarily—older data may degrade performance. Focus on adding information, not just volume.

    31. How to handle label noise?
    Use expert-reviewed subsets, triangulation with multiple labels, and accept some noise with large datasets.

    MODULE 6: Best Practices, Teams, and Launching ML in Healthcare #

    32. What is the Output-Action Pair (OAP)?
    A framework linking model prediction to clinical action (e.g., sepsis alert → early intervention).

    33. Why multidisciplinary teams matter?
    Clinicians, statisticians, ML engineers, IT staff, and ethicists ensure models are usable, reliable, ethical.

    34. What roles are essential in clinical ML teams?

    • Data scientist (features, metrics)
    • ML engineer (model building & deployment)
    • Statistician (study design, validation)
    • Healthcare IT (integration)
    • Domain expert (clinical relevance)

    35. What governance and ethics concerns exist?

    • Risk of re-identification
    • Informed consent
    • Algorithmic bias
    • Health equity

    36. What is automation bias?
    Over-trusting ML systems leads to reduced vigilance—counteract with calibrated risk scores and human oversight.

    37. Will ML replace healthcare jobs?
    Not fully—ML will augment routine tasks but jobs requiring empathy, judgment, and interaction remain essential.

    FINAL TAKEAWAYS #

    • Simpler models are often more deployable.
    • Be skeptical of great metrics—investigate false positives/negatives.
    • Use external datasets and real-world data for true performance estimates.
    • Multidisciplinary feedback loops are essential for trust and improvement.
