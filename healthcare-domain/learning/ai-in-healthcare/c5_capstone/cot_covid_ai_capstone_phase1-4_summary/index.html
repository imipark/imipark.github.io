    The Stanford AIMI Center lab accesses a large COVID-19 dataset published in a high-impact journal. It includes 30,000 exams from 21,000 patientsâ€”each exam has a CXR image and EHR data.

    Two ML projects are born:

    1. CXR-based Deep Learning Model â€“ Predict COVID presence from x-rays.
    2. EHR-based Logistic Regression Model â€“ Predict intubation risk.

    Challenges Identified:

    • Severe class imbalance (90% COVID-negative, 10% COVID-positive).
    • Image data is large (3000x3000 px), requiring resizing or high-memory GPUs.
    • EHR data has missing values and symptom columns are binary.
    • There’s also a similar, larger EHR dataset (40,000 non-COVID respiratory illness exams) that can be useful for pretraining.

    Project 1: CXR-Based Classifier #

    What happened?

    • ResNet-50 was used on resized 224x224 images.
    • Data augmentation: random zooming.
    • Random data splits were used, not patient-level.
    • Training loss showed minimal improvement â†’ model underfit or flawed preprocessing.

    Identified issues:

    • Poor performance possibly due to too much downscaling or improper splitting.

    Project 2: EHR-Based Intubation Predictor #

    Findings:

    • Only 3,000 usable EHR records, not 30,000.
    • High sparsity and strange lab value distributions (e.g. D-DIMER, Ferritin).
    • Missing values were common.
    • Logistic regression wasn’t feasible unless missing values were addressed.

    ðŸ”§ Phase 3: Model Training, Part 2 #

    Project 1: CXR-Based Classifier #

    Improvements:

    • Data split by patient to prevent leakage.
    • Upsized images to 512x512; architecture adapted accordingly.
    • Augmentation simplified to horizontal flips and light zoom.
    • Oversampling of COVID-positive cases added.

    Problem now:

    • Overfitting. Training loss is much lower than validation loss.
    • Discrepancy between AUROC (low) and accuracy (high) on validation set â†’ possible threshold or label leakage issue.

    Project 2: EHR-Based Predictor #

    Strategy change:

    • Use the 40,000-exam COVID-like dataset for training/validation.
    • COVID dataset now reserved as a test set.
    • Cross-validation (10-fold) used for hyperparameter tuning.
    • Random Forests also explored alongside logistic regression.

    Project 1: CXR-Based Classifier #

    New changes:

    • Used dropout (0.5) and added random rotations to improve generalization.
    • Two models trained:
      • Model A: Early stopped on best AUROC.
      • Model B: Early stopped on best validation loss.

    Project 2: EHR-Based Predictor #

    Focus:

    • COVID-like training + COVID test set approach is working well.
    • Now evaluating the precision-recall curve to choose an operating point (threshold).
    • Need help picking a threshold that fits the triage use case.
