    [Overview] Data-Centric AI #

    Q1: What is the overall motivation behind Data-Centric AI? #

    • Traditional AI focused on improving models given fixed data.
    • Data-Centric AI focuses on improving the data itself (quality, coverage, relevance, fairness) because data is the dominant factor in ML system success.

    Q2: How does dataset curation fit into Data-Centric AI? #

    • Dataset creation and curation is the foundation.
    • It asks: Are we collecting the right examples? Is the labeling accurate and representative?
    • Techniques: thoughtful sampling, careful label aggregation (e.g., majority vote, Dawid-Skene, CROWDLAB).

    Q3: Once data is collected, what problems commonly arise? #

    • Label errors: wrong or inconsistent labels hurt model reliability.
    • Class imbalance: rare classes get ignored.
    • Outliers: strange data points can mislead models.
    • Distribution shift: training and deployment data differ over time.

    Q4: How do we handle such data issues? #

    • Confident Learning identifies and prunes likely label errors systematically.
    • Active learning selectively labels the most informative examples.
    • Core-set selection compresses large datasets without losing essential information.
    • Outlier detection techniques (e.g., Isolation Forests, Autoencoders) remove harmful anomalies.

    Q5: How is model evaluation impacted by a data-centric view? #

    • Not just global accuracy: evaluation must check underperforming slices (e.g., specific demographics).
    • Must detect model bias, brittle subpopulations, and errors that standard metrics would miss.
    • Influence functions (like Data Shapley) help prioritize which datapoints matter most to fix.

    Q6: How does interpretability tie into Data-Centric AI? #

    • If features themselves are messy or unintuitive, the model can’t be interpretable.
    • Interpretable feature engineering (with users in the loop) enhances both trust and model robustness.
    • Example tools: Ballet, Pyreal, Mind-the-Gap feature frameworks.

    Q7: Why is privacy and security essential in the Data-Centric mindset? #

    • ML models often memorize sensitive training data unintentionally.
    • Threats include:
      • Membership inference attacks (detect if a datapoint was in training)
      • Data extraction attacks (leak sensitive records)
    • Solutions involve differential privacy (DP), model output smoothing, and rigorous threat modeling.

    Q8: How does LLM-specific curation amplify Data-Centric principles? #

    • Data curation for LLMs is even more critical because LLMs memorize and generalize at a massive scale.
    • Techniques include:
      • Fine-tuning with synthetic but high-quality generated datasets.
      • Uncertainty quantification to filter low-confidence generations.
      • Applying both human and LLM evaluators to assess outputs robustly.

    Q9: What is the grand cycle of a data-centric AI system? #

    1. Curate and clean the initial dataset.
    2. Train simple models to uncover data issues.
    3. Fix data issues (label noise, bias, outliers).
    4. Improve feature quality for interpretability and usability.
    5. Evaluate deeply, especially for subpopulations.
    6. Secure the model and data against privacy attacks.
    7. Iterate: new data, new shifts, new repairs — endlessly evolving.

    Q10: Final Takeaway #

    • Data is the first-class citizen in AI systems.
    • Success in Data-Centric AI requires mastering curation, quality control, evaluation, privacy protection, and interpretability — before throwing bigger models at the problem.
